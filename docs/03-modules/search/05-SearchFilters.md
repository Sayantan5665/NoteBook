# 05 — Search Filters

---

## 1. Purpose

This document outlines the conceptual framework for Search Filters. Filters act as scoping constraints that narrow down the result set returned by a Search Query.

## 2. Filter Concepts

Filters are conceptual predicates applied to the search index during query execution. They restrict the domains or metadata properties that the query interrogates.

### 2.1 Available Filters (Examples)
- **Folder:** Restrict results to a specific Folder UUID (or its descendants).
- **Tags:** Require results to possess a specific Tag UUID.
- **Attachment Types:** Restrict results to Notes containing specific MIME types (e.g., Images, PDFs).
- **Temporal Dates:** Filter by `Created Date` or `Modified Date` ranges.
- **Boolean Flags:**
  - `Has Attachments`
  - `Has OCR`
  - `Has Wiki Links`
  - `Has Backlinks`
  - `Favorites`
  - `Archived`

### 2.2 Future Filters
- **AI Metadata:** Filtering by conceptual topics generated by future AI modules.
- **Custom Properties:** Filtering by user-defined key-value pairs (frontmatter).

## 3. Filter Composition

### 3.1 Filter Combinations
- Filters can be logically combined using `AND` or `OR` (e.g., `Tags: #urgent` AND `Has Attachments: true`).
- A Search Query combines a primary text string with an array of active Filters.

## 4. Filter Validation

- **Integrity Checks:** The system must validate that the requested Filter exists and that the supplied arguments are mathematically valid (e.g., ensuring a Date Range Filter does not have an End Date preceding the Start Date).
- **Graceful Failure:** If a user supplies an invalid Filter via an API or malformed Saved Search, the module drops the invalid filter and executes the rest of the query, rather than crashing.

## 5. Business Rules

- **Narrowing Scope:** Filters exist exclusively to refine and narrow Search Results.
- **Non-Destructive:** Filters NEVER change indexed data or canonical Note data. They simply instruct the query engine to ignore specific branches of the index.

## 6. Acceptance Criteria

- Applying a `Has Attachments` filter to a blank query successfully returns a list of all Notes that contain at least one Attachment UUID.
- Removing a filter instantly expands the Search Results back to their broader scope without requiring a reindex of the database.
